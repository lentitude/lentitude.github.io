基于 Github Page + Hexo 搭建好博客之后, 需要使用 hexo 命令来完成文章的新建, 预览和部署等操作, 这里总结了常见命令的写法和注意事项

命令的简写

hexo n = hexo new
hexo p = hexo publish
hexo g = hexo generate
hexo s = hexo server
hexo d = hexo deploy

博客的生成顺序

  1. 执行 hexo new 'title' 在 _post 目录下生成 title.md 文件
  2. 博客写好后, 执行hexo g 生成静态页面
  3. 执行hexo d 部署到 Github

可以将 hexo g和hexo d 进行合并为 hexo g -d， 需要注意 hexo g -d 和 hexo d -g 效果等同，-d 表示文件生成后立即部署网站, -g 表示部署之前预先生成静态文件, 所以先生成静态文件再部署的顺序始终是保存一致的。

一般都是通过这种手动的方式设置摘要, 毕竟使用自动摘要的方式, 会出现乱断句的问题

草稿的用法

很多情况下，你无法在创建之后完成一篇博客的编写, 也不想花额外的精力来管理为完成的博客, 这时候就可以使用草稿功能, 当然你也可以养成这个习惯。

hexo new draft 'draft'

会在 source/_drafts 目录下生成一个 draft.md 文件, 但是该文件不会不会被访问到。

如果需要预览草稿文章, 可以通过更改配置文件 render_drafts: true , 或者通过 hexo server --drafts 来启动 server。

如果需要将草稿转为正式文章或者界面, 可以使用命令 hexo publish [layout] <filename>

服务器

hexo server 监视文件变动并自动更新, 无须重启服务器
hexo server -s 静态模式
hexo server -p 5000 更改端口
hexo server -i 192.168.1.1 自定义IP
hexo clean 清楚缓存
hexo g 生成静态网页
hexo d 开始部署
